## Claude-Context MCP 使用规则

**何时使用语义搜索 (search_code):**
- 理解代码架构和组件关系时（如"optimistic execution 是如何实现的"）
- 查找特定功能的实现位置（如"conflict detection 相关代码"）
- 寻找类似模式或最佳实践（如"其他地方如何处理并行执行"）
- 在修改代码前，先搜索相关实现以了解上下文
- 回答"哪里"、"如何"、"为什么"类型的问题

**搜索工作流:**
1. 先用 `search_code` 进行语义搜索，获取相关代码片段
2. 基于搜索结果使用 `Read` 读取完整文件
3. 理解上下文后再进行修改

**搜索最佳实践:**
- 使用自然语言描述要查找的内容（中文或英文都可以）
- 可以指定 `extensionFilter` 限制文件类型（如 `['.rs']`）
- 使用 `limit` 参数控制返回结果数量（默认10，最多50）
- 搜索前确保代码库已完成索引（使用 `get_indexing_status` 检查）

**何时不使用语义搜索:**
- 已知确切文件路径时，直接用 `Read`
- 查找特定函数名或类名时，用 `Grep` 或 `Glob` 更快
- 简单的文件查找，用 `Glob` pattern matching
